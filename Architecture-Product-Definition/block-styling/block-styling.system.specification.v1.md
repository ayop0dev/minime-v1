# Block Styling System Specification V1

## Purpose

This document defines the Block Styling System used by Minime.

The Block Styling System is responsible for the visual appearance of individual block instances.

It allows blocks to inherit Theme styles while supporting block-specific visual customization.

---

# Philosophy

Themes define:

```text
Default Appearance
```

Block Styling defines:

```text
Individual Appearance
```

Rule:

```text
Theme
Provides Defaults

Block Styling
Provides Overrides
```

---

# Scope

The Block Styling System is responsible for:

* Block visual styling
* Style inheritance
* Style overrides
* Style resolution
* Block style validation
* Block-specific visual customization

---

# Out Of Scope

The Block Styling System is not responsible for:

* Block content
* Block ordering
* Block creation
* Block deletion
* Theme definitions
* Theme catalogs
* Theme lifecycle
* Rendering
* Layout
* Analytics

---

# Core Responsibility

The system controls:

```text
How A Specific Block Looks
```

It does not control:

```text
What The Block Contains
```

---

# Relationship With Themes

Themes provide default visual values.

Example:

```text
Button Radius
Button Colors
Button Border
Button Shadow
```

Blocks inherit these values automatically.

---

# Relationship With Blocks

Each block may contain:

```text
Block
├─ Content
├─ Settings
└─ Style
```

The Style section belongs to the Block Styling System.

---

# Relationship With Rendering

The Renderer never reads Theme defaults directly.

The Renderer consumes:

```text
Resolved Block Style
```

Generated by the Block Styling System.

---

# Styling Levels

Block styling exists at two levels.

## Theme Default Style

Provided by the active Theme.

Example:

```text
Button Radius = 16px
```

---

## Block Style Override

Provided by a specific block.

Example:

```text
Button Radius = 24px
```

For that block only.

---

# Inheritance Model

Blocks inherit Theme values automatically.

Flow:

```text
Theme Style
        ↓
Block Style
        ↓
Resolved Style
```

---

# Override Model

When a block defines a value:

```text
Block Value
Overrides
Theme Value
```

Example:

```text
Theme Radius = 16px

Block Radius = 30px

Result = 30px
```

---

# Style Ownership

Theme owns:

```text
Default Styles
```

Block owns:

```text
Style Overrides
```

This separation is mandatory.

---

# Style Resolution

Final visual styles are generated through style resolution.

Flow:

```text
Theme Defaults
        ↓
Block Overrides
        ↓
Constraint Validation
        ↓
Resolved Style
```

Only resolved styles may be rendered.

---

# Supported Blocks

The Block Styling System applies to all visual blocks.

Examples:

```text
Image
Avatar
Name
Bio
Button
Social Icons
Divider
Text
```

Future block types automatically participate in the styling system.

---

# Profile Scope

Block styling operates per block instance.

Rule:

```text
One Block
=
One Style Context
```

Each block may have unique styling.

---

# Theme Independence

Block Styling does not modify Theme definitions.

Changing a block style:

```text
Does Not Change Theme
```

Changing a Theme:

```text
May Affect All Inheriting Blocks
```

---

# Constraint Enforcement

All style values must comply with Theme constraints.

Invalid style values are rejected.

---

# V1 Principles

```text
Themes Provide Defaults

Blocks Provide Overrides

Blocks Own Style Overrides

Themes Own Default Styles

Styles Are Resolved Before Rendering

Block Styling Does Not Affect Content

Block Styling Does Not Affect Layout

Every Block Has Its Own Style Context
```
