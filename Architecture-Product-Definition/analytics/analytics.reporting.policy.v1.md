# Analytics Reporting Policy V1

## Status

Approved

---

# Purpose

The Reporting Layer is responsible for presenting analytics metrics to Account owners.

Reports transform aggregated metrics into readable information.

Reports do not generate events.

Reports do not calculate metrics.

---

# Core Principle

```text
Reporting Displays Metrics
```

Reporting consumes metrics generated by the Aggregation Layer.

---

# Architecture Position

```text
Events
↓
Aggregation
↓
Metrics
↓
Reporting
```

---

# Report Ownership

Reporting owns:

```text
Metric Presentation
Filtering
Ordering
Display Structure
```

Reporting does not own:

```text
Event Creation
Metric Calculation
Data Collection
```

---

# V1 Report Types

Supported reports:

```text
Profile Analytics Report
Link Analytics Report
```

Only.

---

# Profile Analytics Report

Purpose:

```text
Provide Overall Profile Performance
```

Displays:

```text
Total Views
Total Clicks
CTR
Device Category Breakdown
```

---

# Profile Analytics Example

```text
Views: 1,250

Clicks: 180

CTR: 14.4%

Device Categories:
Mobile: 82%
Desktop: 15%
Tablet: 3%
```

---

# Link Analytics Report

Purpose:

```text
Provide Link Performance Data
```

Displays:

```text
Link Label
Destination URL
Click Count
Rank
```

---

# Link Analytics Example

```text
1. YouTube
120 Clicks

2. WhatsApp
80 Clicks

3. LinkedIn
25 Clicks
```

---

# Link Identity Rule

Reports must use:

```text
Out Link Identity
```

for analytics.

Reports must not merge:

```text
Different Out Links
```

even if they originated from the same Block.

---

# Time Range Filtering

Supported filters:

```text
Today
Last 7 Days
Last 30 Days
All Time
```

Only.

---

# Default Time Range

Default:

```text
Last 30 Days
```

unless otherwise specified by the user interface.

---

# Sorting Rules

Link reports may be sorted by:

```text
Click Count
```

Highest first.

Default:

```text
Descending
```

order.

---

# Empty State Rules

If no analytics data exists:

```text
Display Zero Values
```

Example:

```text
Views: 0
Clicks: 0
CTR: 0%
```

No estimation occurs.

---

# Historical Integrity

Analytics events are append-only.

Existing events are never modified.

Historical analytics are never rewritten.

Reports must reflect:

```text
Actual Stored Events
```

Only.

Reports must not:

```text
Predict
Estimate
Forecast
```

future values.

---

# Archived Out Links

Archived Out Links may appear in reports while their analytics data exists.

Example:

```text
Archived Link
↓
Historical Click Data Visible
```

until retention expires.

---

# Purged Out Links

Purged Out Links are not reported.

Reason:

```text
Data No Longer Exists
```

---

# Data Consistency

Reports must be consistent with:

```text
Aggregation Output
```

Reports may never modify:

```text
Metrics
Events
```

---

# V1 Visualization Scope

Reports may display:

```text
Numbers
Percentages
Ranked Lists
Simple Breakdowns
```

Only.

---

# V1 Non-Goals

Not supported:

```text
Funnels
Heatmaps
Behavior Flows
Cohorts
Forecasts
Comparative Intelligence
Benchmarks
```

---

# Audience

Reports are intended for:

```text
Account Owners
```

Only.

Reports are not public.

Reports are not shared with visitors.

---

# Design Goal

The Reporting Layer should answer:

```text
How is my profile performing?
Which links receive the most clicks?
What is my CTR?
What device categories are visitors using?
```

using the simplest possible reporting experience.

Reports should remain:

```text
Clear
Minimal
Accurate
Actionable
```
